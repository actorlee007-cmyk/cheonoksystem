# [티스토리용 / 노코드 AI 자동화 튜토리얼] n8n + Claude Code로 블로그 글감 자동 생성 시스템 만들기 (실전 입문)

매일 블로그에 뭘 올릴지 정하는 게 생각보다 큰 부담입니다. "오늘은 뭘
쓰지", "제목은 어떻게 잡지", "마무리는 어떻게 하지"를 매번 처음부터
고민하면 포스팅 빈도가 들쭉날쭉해지고, 빈도가 들쭉날쭉하면
검색엔진/애드센스 심사에도 좋지 않습니다.

이 글에서는 "글의 뼈대(주제·제목·목차·CTA)"를 자동으로 뽑아주는
파이썬 스크립트를 만들고, n8n으로 매일 정해진 시간에 실행되게 하는
구조를 소개합니다. (실제 본문은 사람이 채웁니다 — AI가 글 전체를
대필하는 구조가 아닙니다.)

## 1. 이 자동화가 해결하는 문제

- "오늘 뭐 쓰지" 고민 시간을 줄인다
- 카테고리를 미리 정해두고 요일별로 분산해서 콘텐츠 다양성을 확보한다
- 글의 형식(제목/도입부/목차/CTA)을 일관되게 유지해서 SEO와 가독성을
  동시에 챙긴다

## 2. 준비물

- Python 3.x가 설치된 서버 또는 PC (n8n이 돌아가는 환경)
- n8n (클라우드 또는 셀프호스팅 둘 다 가능)
- (선택) 네이버 오픈 API 키 — developers.naver.com에서 무료 발급, 검색
  API로 요즘 상위노출되는 제목을 참고용으로 가져올 때 사용
- ⚠️ API 키는 절대 코드/Git 저장소/공개 워크플로 JSON에 하드코딩하지
  않고, n8n Credentials 또는 Variables에만 저장합니다

## 3. 단계별 설정

**3-1. 글감 생성 스크립트 준비**

1. "주제 카테고리"를 미리 정의합니다 (예: 운영일지, 튜토리얼, 리포트
   해설, 후기 — 4가지).
2. 카테고리별로 "제목 템플릿 - 도입부 - 목차 - CTA" 구조를 코드로
   작성합니다.
3. (선택) 검색 API로 같은 키워드의 상위노출 제목을 가져와 참고 신호로
   추가합니다.
4. API가 막히거나 키가 없어도 기본 템플릿으로 결과가 나오도록
   fallback을 반드시 만듭니다. **자동화는 절대 멈추면 안 됩니다.**

**3-2. n8n 워크플로 구성**

1. Schedule Trigger (예: 매일 오전 9시)
2. Execute Command 노드 → 위 스크립트 실행
3. 결과 CSV/마크다운 파일을 읽어서 Slack/이메일/Google Sheet로 전달
4. (자동 게시 아님) 사람이 받아서 검토 후 직접 블로그에 게시

**3-3. 산출물 형식**

- JSON: 전체 데이터 (디버깅/재사용용)
- CSV: 콘텐츠 큐 (상태 관리용 — `SEND_READY` → `POSTED`)
- Markdown: 바로 복붙 가능한 글 초안

## 4. 자주 발생하는 오류와 해결

- **API 키 미설정** → `NO_API_KEY`로 기록하고 오프라인 템플릿으로
  대체합니다 (멈추지 않음).
- **API 호출 실패(네트워크/권한)** → 에러 메시지를 그대로 기록하고
  오프라인 템플릿으로 대체합니다. "성공"으로 거짓 보고하지 않는 게
  핵심입니다.
- **같은 주제 반복** → 카테고리를 요일별로 분산 배치(예: 월/목
  운영일지, 화/금 튜토리얼)해서 다양성을 확보합니다.

## 5. 다음 단계로 확장하는 아이디어

- 카테고리를 늘려서(제품 리뷰, Q&A 등) 더 다양한 글감을 확보한다
- 게시 후 URL을 다시 콘텐츠 큐에 기록해서 "어떤 주제가 반응이
  좋았는지" 누적 데이터로 만든다
- 같은 소재를 네이버 블로그/티스토리에 톤만 다르게 변형해서 배포한다
