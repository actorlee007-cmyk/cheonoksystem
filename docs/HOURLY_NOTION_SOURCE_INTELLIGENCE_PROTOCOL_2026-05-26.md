# JOS Hourly Notion Source Intelligence Protocol — 2026-05-26

## Trigger
CEO instructed that the system must check Notion every hour for newly added links, then deeply analyze all sources such as channels, TikTok, Instagram, Threads, X, websites, tools, systems, philosophies, operating models, and sales/distribution structures, and update JOS accordingly.

## Core Rule
A link is not a bookmark.
A link is a system signal.

Every new link must be decomposed and merged through:

```text
Source Intake
→ Platform Identification
→ Essence / Context Diagnosis
→ Tool and System Decomposition
→ Operating Model Classification
→ Philosophy / Positioning Analysis
→ Canon Filter
→ Top-1 Perspective Council
→ CASH CHAIN Mapping
→ JOS Merge Decision
→ Evidence Record
→ Next Patch
```

## Hourly Scan Scope
The hourly scanner must check:

1. Notion newly added links
2. Hjarni notes, if used as secondary memory
3. Google Drive recent documents, if relevant
4. Airtable Content Queue / External Automation Matrix / Hourly Source Intake
5. GitHub docs or issues only when a source was already promoted to canon/patch

## Source Types
Each new item must be classified as one or more:

- YouTube channel
- TikTok account/video
- Instagram account/post/story/reel
- Threads account/post
- X account/post/thread
- Website / landing page
- SaaS / app / tool
- AI agent / automation system
- Affiliate / ads funnel
- Subscription product
- Direct operation business
- Sales network / distribution model
- Community / audience system
- Research / philosophy / ideology / positioning source
- Competitor / benchmark

## Required Decomposition
For every source, capture:

1. What is the surface content?
2. What is the hidden operating system?
3. Is it direct operation, tool-stack business, agency/service, sales network, content engine, affiliate funnel, subscription system, or mixed?
4. What tools or workflows are likely being used?
5. What is the customer psychology?
6. What is the positioning / philosophy / worldview?
7. How does it create attention?
8. How does it convert attention into lead/payment/subscription?
9. What can JOS absorb without copying?
10. What must be blocked due to risk, privacy, legal, false claim, or copy-risk?

## Canon Filter
Before merging, every source must pass:

- North Star alignment: EXPOSURE -> LEAD -> PAYMENT -> DELIVERY -> REVIEW -> SUBSCRIPTION
- JOS master canon fit
- PAPER_ONLY safety when related to markets
- Final Veto
- Non-regression memory
- Evidence gate

The hourly run must not produce vague summaries.

1. New sources found
2. Sources skipped as duplicate

## Important Limitations
Some platforms may block scraping or require login.
If a source cannot be accessed, the system must not guess.
It must mark the item as HOLD with the reason:
- login required
- unavailable
- private account
- rate-limited
- unsupported platform
- insufficient evidence

## Final Canon Line
JOS must inspect new links as operating systems, not content. Every source must be decomposed into tools, business model, sales/distribution route, philosophy, customer psychology, and JOS absorption path, then merged only through canon filter, top-1 perspective council, evidence gate, and Final Veto.
