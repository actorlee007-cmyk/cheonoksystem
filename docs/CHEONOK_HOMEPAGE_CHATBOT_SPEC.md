# CHEONOK 홈페이지 챗봇 정본 v1

## 0. 목적

CHEONOK 홈페이지 챗봇은 단순 고객상담창이 아니다.

역할은 아래 5개다.

```text
1. 방문자의 예상 질문을 즉시 해결한다.
2. 무료진단/구독/결제까지 자연스럽게 안내한다.
3. 방문자가 해결하고 싶은 불편함을 수집한다.
4. 수집된 불편함을 Pain to Product Engine으로 보낸다.
5. 구독 상품 개선과 광고 카피 개선의 원천 데이터로 사용한다.
```

---

## 1. 챗봇이 해결해야 하는 방문자 불편함

방문자는 홈페이지에서 아래 질문을 즉시 해결하고 싶어 한다.

```text
이게 뭐 하는 서비스인가?
내가 왜 써야 하나?
어떤 자료를 넣으면 되나?
결과로 뭘 받나?
가격은 얼마인가?
무료로 먼저 써볼 수 있나?
결제하면 바로 쓸 수 있나?
핸드폰에서도 되나?
내 자료가 안전한가?
주식 자동매매를 바로 해주는가?
나 같은 사람에게 맞나?
```

이 질문에 답하지 못하면 방문자는 이탈한다.

---

## 2. 챗봇 기본 포지션

챗봇의 첫 문장은 기능 설명이 아니라 방문자의 행동을 유도해야 한다.

```text
좋은 링크나 아이디어가 있는데 어떻게 돈 되는 구조로 바꿀지 막히셨나요?
링크, 아이디어, 불편함 중 하나를 보내주세요.
CHEONOK이 핵심 신호와 다음 실행을 정리해드립니다.
```

---

## 3. 챗봇 메뉴 구조

첫 화면 버튼은 6개로 고정한다.

```text
1. 이 서비스가 뭔가요?
2. 무료진단 받기
3. 구독/가격 보기
4. 내가 가진 링크 분석하기
5. 해결하고 싶은 불편함 남기기
6. 주식 시스템 안전 기준 보기
```

## 4. 예상 질문 답변 정본

### Q1. CHEONOK은 무엇인가요?

```text
CHEONOK은 SNS·유튜브·뉴스·깃허브·이미지·아이디어를 받아서
사람들이 왜 반응하는지,
어떤 불편함이 숨어 있는지,
돈이 되는 구조가 있는지,
콘텐츠·상품·홈페이지·자동화로 어떻게 바꿀지 정리해주는 AI 시장 신호 분석 OS입니다.
```

### Q2. 무엇을 넣으면 되나요?

```text
유튜브 링크, 틱톡 링크, 스레드 글, 뉴스, 깃허브, 이미지 캡처, 아이디어 메모, 불편한 업무 설명을 넣을 수 있습니다.
```

### Q3. 결과로 무엇을 받나요?

```text
핵심 신호, 사람들이 반응한 욕망, 불편함, 위험 요소, PASS/HOLD/BLOCK 판정, 콘텐츠 아이디어, 상품화 방향, 다음 실행 3개를 받습니다.
```

### Q4. 무료로 써볼 수 있나요?

```text
첫 신호 분석 또는 샘플 리포트는 무료진단으로 제공할 수 있습니다.
무료진단 후 반복 분석, 구독 리포트, 홈페이지/상품화 변환은 유료 구독으로 연결됩니다.
```

### Q5. 구독하면 무엇이 달라지나요?

```text
구독자는 매일 또는 매주 시장 신호 분석, 콘텐츠 아이디어, 상품화 후보, 다음 실행 리포트, CEO 요약 보고서를 받을 수 있습니다.
```

### Q6. 결제는 어떻게 하나요?

```text
결제는 안전한 외부 결제 링크를 통해 진행합니다.
카드번호나 민감한 결제정보는 챗봇에 입력하지 않습니다.
```

### Q7. 주식 자동매매를 바로 해주나요?

```text
아니요. 실전 주문과 자본 투입은 차단되어 있습니다.
현재는 PAPER 데이터 축적, 시뮬레이션, 리포트, 후보 분석 중심입니다.
LIVE_TRADE, CAPITAL_SCALE, KIS_ORDER_GATE는 BLOCKED 상태입니다.
```

### Q8. 내 정보는 안전한가요?

```text
민감한 개인정보, 계좌 비밀번호, 카드번호, 증권사 비밀번호는 입력하지 않습니다.
공개 링크와 사용자가 제공한 아이디어 중심으로 분석합니다.
```

## 5. 챗봇이 반드시 수집해야 할 정보

무료진단/문의 시 아래 5개만 받는다.

```text
1. 이름 또는 닉네임
2. 연락 가능한 이메일/텔레그램/카카오 중 하나
3. 분석하고 싶은 링크 또는 아이디어
4. 지금 가장 불편한 점
5. 원하는 결과: 콘텐츠 / 상품화 / 자동화 / 주식 리포트 / 홈페이지 개선
```

입력 항목이 많으면 전환율이 떨어진다.

---

## 6. 결제 전환 흐름

챗봇은 바로 결제를 강요하지 않는다.

```text
무료진단
↓
샘플 리포트 확인
↓
구독 플랜 추천
↓
결제 링크 안내
↓
결제 후 온보딩 안내
```

결제 버튼 문구:

```text
무료진단 먼저 받기
샘플 리포트 보기
구독 플랜 확인하기
상담 후 결제하기
```

## 7. 가격제/구독제 임시 구조

최종 가격은 시장 반응 후 조정한다.

```text
Free: 샘플/1회 진단
Starter: 월간 소셜 신호 리포트
Pro: 주간/일간 리포트 + 콘텐츠/상품화 제안
CEO: 맞춤형 CEO 보고서 + 시스템 패치 후보
```

챗봇은 방문자 목적에 따라 플랜을 추천한다.

```text
콘텐츠가 필요하면 Starter/Pro
자동화와 상품화가 필요하면 Pro
사업 전체 운영 보고가 필요하면 CEO
```

## 8. Pain to Product 연결

챗봇은 모든 문의를 아래 구조로 변환해야 한다.

```text
방문자 발언
↓
불편함 추출
↓
반복 빈도 추정
↓
대중성 추정
↓
긴급성 추정
↓
행동 수 감소 가능성 판단
↓
상품화 후보 저장
```

이 데이터는 광고 문구와 상품 개선에 다시 사용한다.

## 10. 홈페이지 내 배치

챗봇은 오른쪽 하단 고정 버튼으로 둔다.

섹션별 CTA는 다음과 같다.

```text
Hero: 무료진단 받기
Pain Section: 내 불편함 남기기
Signal Section: 링크 분석하기
Subscription Section: 구독 플랜 물어보기
Safety Section: 주식 시스템 안전 기준 보기
Footer: 문의/제안 남기기
```
