# JOS Tool Connection Priority Matrix — 2026-05-25

## Core Judgment
A world-class JOS/CHEONOK system should not connect random apps just because they exist.
The highest-leverage tools are the ones that close the operating loop:
input -> memory -> execution -> revenue -> delivery -> evidence -> upgrade.

## Priority 0 — Already Critical Internal Connectors
These should remain the core operating spine:
- GitHub: code, issues, patches, runbooks, evidence of changes
- Google Drive: canon, reports, docs, sheets, archives
- Airtable: CRM, task queue, content calendar, tool registry, ledgers
- Stripe: payment links, products, subscriptions, payment verification
- PayPal: invoice and alternate payment path
- Vercel: web, API, landing pages, subscription portal
- Gmail: delivery, support, outreach, report sending
- Canva: thumbnails, report covers, proposal visuals, social assets
- Figma: diagrams, product UI, pitch decks, design system

## Priority 1 — Tools From The Current App Screen
Based on the visible tool list, the highest-leverage candidates are:

Role: Canon and topic graph layer.
Why: JOS needs a graph of concepts, decisions, rules, failures, patches, and business-line connections.
Routing:

Why: Useful only if it improves lead, customer, product, or investor pipeline. Airtable remains primary until this proves stronger.

### 5. Adalo or Dazzy

### 6. Goodnotes
Role: Document/notes insight extraction.
Why: Useful only if handwritten or PDF-based notes become part of the JOS knowledge flow.
Routing:

## PASS / HOLD / BLOCK Rule
PASS: tool closes a current revenue, evidence, delivery, or data loop.
HOLD: tool is interesting but duplicates an existing tool or lacks account/access readiness.
BLOCK: tool increases manual work, exposes sensitive data, creates legal/payment risk, or has no business-line route.

## Final Canon Line
The 100x tool stack is not the largest stack.
It is the smallest connected stack that makes JOS observe, remember, decide, execute, collect revenue, deliver value, measure evidence, and upgrade itself.

Current status: autonomous learning and PAPER data accumulation.
