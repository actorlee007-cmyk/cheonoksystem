# JOS Top-1 Organic Role Council Protocol — 2026-05-26

## Core Correction
The current JOS/CHEONOK system must not behave as a collection of separate tools, chats, agents, documents, and reports.
Every system must become a role node inside one organic problem-solving council.

The correct principle is:
- One problem enters.
- Multiple top-1 roles inspect it.
- Each connected system contributes only its strongest function.
- The Orchestrator merges the output.
- Final Veto blocks unsafe or unverified claims.
- Evidence is written to ledgers.
- The next patch is created.

## Required Organic Loop
Every issue must follow this loop:

1. CLAIM
What is being claimed or requested?

2. CHECK
Which connected systems can verify the claim?

3. SELF_QUESTION
What would a top-1 operator, programmer, problem solver, revenue architect, security reviewer, and evidence auditor ask?

4. ROOT_CAUSE
What is the actual bottleneck?

5. ROLE_ROUTING
Which role node owns each part?

6. PATCH
What artifact, code, ledger, template, or workflow must change?

7. RETEST
How do we prove the patch works?

8. EVIDENCE
Which ledger, file, URL, record, or report proves it?

9. FINAL_VETO
Is anything unsafe, unverified, or not aligned with canon?

10. NEXT_PROOF
What is the next proof required before calling it complete?

## Operating Rule

Every meaningful issue must be resolved by role collision:
